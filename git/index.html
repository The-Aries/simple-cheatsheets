Git Cheatsheet

Description

This page is a practical cheatsheet for common Git commands. You can fill your placeholders, apply them, and copy commands with your own values. It is not a full Git tutorial. For deeper learning, see the official Git documentation.

Placeholders

Set once, apply to command lines, then copy ready-to-run commands.

Your GitHub username, usually visible on your profile page.

Your GitHub login or commit email used for attribution.

The branch name you use locally for active work.

The branch name tracked on the remote repository.

The remote alias, usually something like origin.

Repository URL copied from the GitHub repository page.

Key Concepts

Local repo lives on your machine. Remote repo lives on GitHub.

Git keeps local and remote repositories synchronized in both directions.

Branch is a work line, remote is the linked server target, and sync covers fetch, pull, and push.

These placeholders customize identity, branch names, remote alias, and repository location in command examples.

Workflow Overview

A common Git workflow usually starts with setup, then daily local-to-remote sync, then branch work, inspection, and integration. The sections below are arranged in that typical usage order, with local and remote repository synchronization as the main thread.

  1. Setup and Repository

    Set identity and connect local repo to remote repo.

  2. Save and Sync

    Stage, commit, fetch, pull, and push as the daily sync loop.

  3. Branching

    Create and switch branches to isolate work safely.

  4. Inspect

    Use status, diff, and log to verify current state.

  5. Fix and Integrate

    Restore, reset, and merge to fix and finalize.

1. Setup and Repository

1.1 config

Set user identity and useful defaults before daily work.

CommandPurposeCopy
git config --global user.name "githubUsername"Set commit author name globally.
git config --global user.email "githubEmail"Set commit email globally.
git config --global init.defaultBranch "localBranchName"Set default branch name for new repositories.

Description

`--global` applies settings to all local repositories on this machine. More flags and advanced combinations are available in the official git-config documentation.

1.2 clone

Create a local repository from an existing remote source.

CommandPurposeCopy
git clone "repoUrl"Clone default branch to local machine.
git clone -b "remoteBranchName" "repoUrl"Clone and checkout a specific remote branch.

Description

Use `-b` when a non-default branch is your starting point. For more options, see the official git-clone documentation.

1.3 init

Start Git tracking in an existing local directory.

CommandPurposeCopy
git initInitialize repository with default branch.
git init -b "localBranchName"Initialize repository with explicit branch name.

Description

Use explicit branch naming when your team standard differs from machine defaults. More options are in the official git-init documentation.

1.4 remote

Attach, inspect, and update remote repository links.

CommandPurposeCopy
git remote add "remoteName" "repoUrl"Add a named remote to the local repository.
git remote -vList configured remotes and URLs.

Description

`add` creates the link and `-v` verifies it. Additional remote workflows are in the official git-remote documentation.

2. Save and Sync

2.1 add

Move working-tree changes into staging area.

CommandPurposeCopy
git add .Stage all changed files in current directory.
git add "file.txt"Stage a specific file only.

Description

Use targeted add for safer commits and full add for broad checkpoints. More flags are available in the official git-add documentation.

2.2 commit

Record staged changes as a local history point.

CommandPurposeCopy
git commit -m "your message"Create commit with one-line message.
git commit --amend -m "updated message"Update most recent commit message/content.

Description

Use amend only before pushing when you need to refine the latest local commit. Full reference options are in the official git-commit documentation.

2.3 fetch

Download remote updates without changing working branch.

CommandPurposeCopy
git fetchFetch from default remotes.
git fetch "remoteName"Fetch from a specific remote explicitly.

Description

Fetch first when you want review control before any merge or rebase happens locally. For additional combinations, see the official git-fetch documentation.

2.4 pull

Fetch and integrate remote changes into current local branch.

CommandPurposeCopy
git pullPull from tracked upstream branch.
git pull "remoteName" "remoteBranchName"Pull a specific remote branch explicitly.

Description

Explicit pull is safer in multi-branch repositories with different tracking targets. For more pull modes, see the official git-pull documentation.

2.5 push

Publish local commits to the remote repository.

CommandPurposeCopy
git pushPush to tracked upstream branch.
git push "remoteName" "localBranchName":"remoteBranchName"Push local branch to an explicit remote branch.

Description

Use explicit branch mapping when local and remote branch names differ. More push flags are in the official git-push documentation.

3. Branching

3.1 branch

Create and inspect branch references.

CommandPurposeCopy
git branch "localBranchName"Create a new local branch.
git branch -aList local and remote branches.

Description

Use branch listing before switching to avoid mistakes in similar names. Extended branch options are in the official git-branch documentation.

3.2 switch

Move between branches using modern command syntax.

CommandPurposeCopy
git switch "localBranchName"Switch to an existing local branch.
git switch -c "localBranchName"Create and switch in one command.

Description

Prefer `switch` for clarity when branch movement is your only action. More variants are in the official git-switch documentation.

3.3 checkout

Use legacy checkout flows for branch and file operations.

CommandPurposeCopy
git checkout "localBranchName"Switch to an existing branch.
git checkout -b "localBranchName"Create and switch branch in one step.

Description

Checkout remains useful for older scripts and mixed branch/file workflows. For full syntax, see the official git-checkout documentation.

4. Inspect

4.1 status

Get quick visibility into local repository state.

CommandPurposeCopy
git statusShow staged, unstaged, and untracked files.
git status -sbShow compact branch-aware summary.

Description

Compact status helps when you need branch and change snapshots quickly. More output formats are in the official git-status documentation.

4.2 diff

Inspect exact code differences before and after staging.

CommandPurposeCopy
git diffShow unstaged working-tree changes.
git diff --stagedShow staged changes ready to commit.

Description

Switch between unstaged and staged diffs to verify final commit content. Additional diff modes are in the official git-diff documentation.

4.3 log

Read commit history for recent activity and branch context.

CommandPurposeCopy
git log --onelineShow compact commit timeline.
git log --oneline --graph --decorateShow branch graph and decorations.

Description

Use graph view when merges make linear history harder to read. For advanced filters and formats, see the official git-log documentation.

5. Fix and Integrate

5.1 restore

Revert local file changes safely in working tree.

CommandPurposeCopy
git restore "file.txt"Restore one file from current HEAD.
git restore .Restore all unstaged file changes.

Description

Use targeted restore first; full restore is faster but riskier in mixed working trees. More restore forms are in the official git-restore documentation.

5.2 reset

Move branch pointers while controlling staging behavior.

CommandPurposeCopy
git reset --soft HEAD~1Undo commit but keep changes staged.
git reset --hard HEAD~1Discard commit and working-tree changes.

Description

Prefer soft reset for recoverable cleanup; hard reset is destructive. For all reset modes, see the official git-reset documentation.

5.3 merge

Integrate work from one branch into another branch.

CommandPurposeCopy
git merge "localBranchName"Merge local branch into current branch.
git merge --no-ff "localBranchName"Keep explicit merge commit for traceability.

Description

Use `--no-ff` when you want branch-level history to remain visible. Additional merge strategies are in the official git-merge documentation.
