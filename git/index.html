Git Cheatsheet

Description

This page is a practical cheatsheet for common Git commands. You can fill your placeholders, apply them, and copy commands with your own values. It is not a full Git tutorial. For deeper learning, see the official Git documentation.

Placeholders

Set once, apply to command lines, then copy ready-to-run commands.

Your GitHub username, usually visible on your profile page.

Your GitHub login or commit email used for attribution.

The branch name you use locally for active work.

The branch name tracked on the remote repository.

Repository URL copied from the GitHub repository page.

Key Concepts

Local repo lives on your machine. Remote repo lives on GitHub.

Git keeps local and remote repositories synchronized in both directions.

Branch is a work line, remote is the linked server target, and sync covers fetch, pull, and push.

These placeholders customize identity, branch names, and repository location in command examples.

Workflow Overview

A typical Git workflow follows this sequence. These modules show how setup, sync, branching, inspection, and fixes connect around local and remote repositories.

  1. 1. Setup and Repository

    Set identity and connect local repository to remote repository.

  2. 2. Save and Sync

    Save local updates and synchronize with remote branch changes.

  3. 3. Branching

    Create and switch branches for isolated development flow.

  4. 4. Inspect

    Check repository state and history before major operations.

  5. 5. Fix and Integrate

    Restore, reset, and merge to correct or finalize changes.

1. Setup and Repository

1.1 config

Set user identity and useful defaults before daily work.

CommandPurposeCopy
git config --global user.name "githubUsername"Set commit author name globally.
git config --global user.email "githubEmail"Set commit email globally.
git config --global init.defaultBranch "localBranchName"Set default branch name for new repositories.

Description

`--global` applies settings to all local repositories on this machine. Use local config without `--global` when one repository needs different identity values.

1.2 clone

Create a local repository from an existing remote source.

CommandPurposeCopy
git clone "repoUrl"Clone default branch to local machine.
git clone -b "remoteBranchName" "repoUrl"Clone and checkout a specific remote branch.

Description

Use the `-b` option when a non-default branch is your starting point, otherwise clone without options for standard onboarding.

1.3 init

Start Git tracking in an existing local directory.

CommandPurposeCopy
git initInitialize repository with default branch.
git init -b "localBranchName"Initialize repository with explicit branch name.

Description

Use explicit branch naming when your team standard differs from machine defaults.

1.4 remote

Attach, inspect, and update remote repository links.

CommandPurposeCopy
git remote add origin "repoUrl"Add origin remote to local repository.
git remote -vList configured remotes and URLs.

Description

`add` creates the link and `-v` verifies it.

2. Save and Sync

2.1 add

Move working-tree changes into staging area.

CommandPurposeCopy
git add .Stage all changed files in current directory.
git add "file.txt"Stage a specific file only.

Description

Use targeted add for safer commits and full add for broad checkpoints.

2.2 commit

Record staged changes as a local history point.

CommandPurposeCopy
git commit -m "your message"Create commit with one-line message.
git commit --amend -m "updated message"Update most recent commit message/content.

Description

Use amend only before pushing when you need to quickly refine the latest local commit.

2.3 fetch

Download remote updates without changing working branch.

CommandPurposeCopy
git fetchFetch from default remotes.
git fetch originFetch from origin explicitly.

Description

Fetch first when you want review control before any automatic merge happens locally.

2.4 pull

Fetch and integrate remote changes into current local branch.

CommandPurposeCopy
git pullPull from tracked upstream branch.
git pull origin "remoteBranchName"Pull a specific remote branch explicitly.

Description

Explicit pull is safer in multi-branch repositories where tracking targets differ across branches.

2.5 push

Publish local commits to the remote repository.

CommandPurposeCopy
git pushPush to tracked upstream branch.
git push origin "localBranchName":"remoteBranchName"Push local branch to explicit remote branch.

Description

Use explicit branch mapping when local and remote branch names differ.

3. Branching

3.1 branch

Create and inspect branch references.

CommandPurposeCopy
git branch "localBranchName"Create a new local branch.
git branch -aList local and remote branches.

Description

Use branch listing before switching to avoid mistakes in similar branch names.

3.2 switch

Move between branches using modern command syntax.

CommandPurposeCopy
git switch "localBranchName"Switch to an existing local branch.
git switch -c "localBranchName"Create and switch in one command.

Description

Prefer `switch` for clarity when branch movement is the only action you need.

3.3 checkout

Use legacy checkout flows for branch and file operations.

CommandPurposeCopy
git checkout "localBranchName"Switch to an existing branch.
git checkout -b "localBranchName"Create and switch branch in one step.

Description

Checkout remains useful for older scripts and repositories still standardized on this command style.

4. Inspect

4.1 status

Get quick visibility into local repository state.

CommandPurposeCopy
git statusShow staged, unstaged, and untracked files.
git status -sbShow compact branch-aware summary.

Description

Compact status helps when you need a fast branch and change snapshot in one command.

4.2 diff

Inspect exact code differences before and after staging.

CommandPurposeCopy
git diffShow unstaged working-tree changes.
git diff --stagedShow staged changes ready to commit.

Description

Switch between unstaged and staged diffs to verify both preparation and final commit content.

4.3 log

Read commit history for recent activity and branch context.

CommandPurposeCopy
git log --onelineShow compact commit timeline.
git log --oneline --graph --decorateShow branch graph and decorations.

Description

Use graph view when branch merges make linear history harder to interpret.

5. Fix and Integrate

5.1 restore

Revert local file changes safely in working tree.

CommandPurposeCopy
git restore "file.txt"Restore one file from current HEAD.
git restore .Restore all unstaged file changes.

Description

Use targeted restore first; full restore is faster but riskier in mixed-change working trees.

5.2 reset

Move branch pointers while controlling staging behavior.

CommandPurposeCopy
git reset --soft HEAD~1Undo commit but keep changes staged.
git reset --hard HEAD~1Discard commit and working-tree changes.

Description

Prefer soft reset for recoverable cleanup. Hard reset is destructive and should be used with caution.

5.3 merge

Integrate work from one branch into another branch.

CommandPurposeCopy
git merge "localBranchName"Merge local branch into current branch.
git merge --no-ff "localBranchName"Keep explicit merge commit for traceability.

Description

Use `--no-ff` when you want branch-level history to remain visible in the commit graph.
