Git Cheatsheet

Description

This page is a practical cheatsheet for common Git commands. Fill in your placeholders, apply them, then copy commands with your own values. It is not a complete Git guide. For deeper coverage, see the official Git documentation.

Placeholders

Set once, apply to all command lines, then copy ready-to-run commands.

Key Concepts

Local repo is the repository on your machine. Cloud repo is the repository on GitHub.

Git helps you keep local repo and cloud repo in sync so changes can move both ways.

Branch is a line of work, remote is the linked cloud location, and sync means fetching, pulling, and pushing.

This page uses placeholders for branch names, repo URL, username, and email so commands match your own setup.

Workflow Overview

Map each section to a concrete stage in local and cloud collaboration.

1. Setup and Repository

1.1 Configure username

1.1.1 Set your Git identity name

git config --global user.name "githubUsername"

Use this once per machine so commits show your preferred display name.

1.2 Configure email

1.2.1 Set your Git login email

git config --global user.email "githubLoginEmail"

Set this to the email you use for GitHub commit attribution.

1.3 Clone repository

1.3.1 Clone from cloud to local

git clone "repoUrl"

Use this when local repo does not exist yet and you need a fresh copy.

1.4 Initialize repository

1.4.1 Create local repo metadata

git init

Use this when starting a brand-new local project before linking to cloud.

1.5 Add remote

1.5.1 Link local repo to cloud repo

git remote add origin "repoUrl"

Use this after `git init` to attach the local repo to a cloud destination.

2. Save and Sync

2.1 Stage files

2.1.1 Add changed files to staging area

git add .

Use this before commit to include current working-tree changes.

2.2 Commit snapshot

2.2.1 Save staged changes locally

git commit -m "your message"

Use this to create a clear checkpoint before syncing with cloud repo.

2.3 Fetch updates

2.3.1 Download cloud changes without merge

git fetch origin

Use this to inspect incoming work before touching localBranch.

2.4 Pull updates

2.4.1 Sync cloudBranch into localBranch

git pull origin cloudBranch

Use this when local work should include the newest remote changes.

2.5 Push commits

2.5.1 Publish localBranch to cloudBranch

git push origin localBranch:cloudBranch

Use this after local commit is ready to share in the cloud repo.

3. Branching

3.1 Create branch

3.1.1 Create localBranch

git branch localBranch

Use this to define a separate line of work before editing files.

3.2 Switch branch

3.2.1 Move to localBranch using switch

git switch localBranch

Use this in modern Git to change branch context safely.

3.3 Checkout branch

3.3.1 Move to localBranch using checkout

git checkout localBranch

Use this in older workflows or mixed repositories that still rely on checkout.

4. Inspect

4.1 Read status

4.1.1 Inspect current working state

git status

Use this first when you need a quick answer about staged and unstaged files.

4.2 Compare changes

4.2.1 Show line-by-line diff

git diff

Use this before commit to verify exactly what changed.

4.3 Review history

4.3.1 Show compact commit history

git log --oneline

Use this to inspect recent commit flow without opening full commit details.

5. Fix and Integrate

5.1 Restore file

5.1.1 Discard local changes in one file

git restore file.txt

Use this when one file should return to the last committed state.

5.2 Reset commit

5.2.1 Move HEAD back while keeping changes

git reset --soft HEAD~1

Use this when the latest commit message or content needs immediate correction.

5.3 Merge branch

5.3.1 Integrate localBranch into cloudBranch

git merge localBranch

Use this to integrate finished work from one branch into another branch.
