Git Cheatsheet

Description

This page is a practical cheatsheet for common Git commands. You can fill your placeholders, apply them, and copy commands with your own values. It is not a full Git tutorial. For deeper learning, see the official Git documentation.

Placeholders

Set once, apply to command lines, then copy ready-to-run commands.

Your GitHub username, usually visible on your GitHub profile page.

Your GitHub login or commit email used for commit attribution.

The branch name you work on in your local repository.

The corresponding branch name tracked on the remote repository.

Repository URL copied from the GitHub repository page.

Key Concepts

Local repo lives on your machine. Remote repo lives on GitHub.

Git helps local and remote repositories stay synchronized in both directions.

Branch is a line of work, remote is a linked server target, and sync covers fetch, pull, and push.

These placeholders customize commands for your identity, branch names, and repository location.

Workflow Overview

A common Git flow runs in this order. Use it to see where each section belongs and how local and remote repositories stay aligned.

  1. Setup and Repository

    Prepare identity and connect local repository to remote repository.

  2. Save and Sync

    Save local changes, then fetch, pull, and push with remote repository.

  3. Branching

    Create and switch branches for isolated work before integration.

  4. Inspect

    Check status, diffs, and history before further changes.

  5. Fix and Integrate

    Restore, reset, and merge when finishing or correcting work.

1. Setup and Repository

1.1 config

Set your Git identity name

git config --global user.name "githubUsername"

Use this once per machine so commits display your preferred username.

1.2 clone

Clone repository from remote

git clone "repoUrl"

Use this when you need a local copy of an existing remote repository.

1.3 init

Initialize a new local repository

git init

Use this when starting from a local folder that is not yet a Git repository.

1.4 remote

Link local repository to remote repository

git remote add origin "repoUrl"

Use this after `git init` when the local repository needs a remote destination.

2. Save and Sync

2.1 add

Stage changed files

git add .

Use this before commit to include current file changes in the next snapshot.

2.2 commit

Create a local commit snapshot

git commit -m "your message"

Use this to record a checkpoint before sharing work to remote repository.

2.3 fetch

Download remote updates without merge

git fetch origin

Use this to inspect incoming work before changing your current branch.

2.4 pull

Sync remote branch into local branch

git pull origin remoteBranchName

Use this when local work should include latest changes from remote branch.

2.5 push

Publish local branch to remote branch

git push origin localBranchName:remoteBranchName

Use this to send local commits to the mapped remote branch.

3. Branching

3.1 branch

Create a local branch

git branch localBranchName

Use this to create a separate line of work before editing files.

3.2 switch

Switch to a local branch

git switch localBranchName

Use this in modern Git workflows to move between branches.

3.3 checkout

Checkout a branch using legacy command

git checkout localBranchName

Use this when working in repositories that still prefer checkout flow.

4. Inspect

4.1 status

Inspect repository working state

git status

Use this first to understand staged files, unstaged files, and branch status.

4.2 diff

Compare line-level code changes

git diff

Use this before committing to verify exactly what has changed.

4.3 log

Read compact commit history

git log --oneline

Use this for a quick timeline of recent commits.

5. Fix and Integrate

5.1 restore

Discard local changes from a file

git restore file.txt

Use this when one file should return to its last committed state.

5.2 reset

Move HEAD back while keeping changes

git reset --soft HEAD~1

Use this when the latest commit needs correction without losing staged content.

5.3 merge

Integrate another branch into current branch

git merge localBranchName

Use this when you are ready to integrate completed branch work.
