Placeholder Description

Each placeholder is shown as a small entry so the sidebar can jump straight to its meaning.

your-name

your-name

Placeholder for the author or contributor name used in local examples.

your-email

your-email

Placeholder for the email value that would normally be configured in Git identity settings.

origin

origin

Placeholder for the default remote name that points to the main repository location.

main

main

Placeholder for the primary branch name used as the central integration point.

remote-url

remote-url

Placeholder for the repository address that Git uses when talking to a remote server.

Setup

Prototype section for initial configuration and repository creation.

Identity

git config --global user.name "your-name"

Use this once to match local commits with the name and email shown in the values panel.

Email

git config --global user.email "your-email"

Use this when the local machine needs a commit email before you start working.

Repository start

git init

Use this when a new folder needs Git metadata before any files are tracked.

Daily Commands

Everyday work stays in short entries so the page reads like a compact technical note.

Check state

git status

Use this first to see what changed before staging or committing anything.

Stage changes

git add .

Use this once the working tree looks ready and you want to prepare a commit.

Create commit

git commit -m "message"

Use this to record the staged changes with a short message.

Branching

Branch operations are split into small entries so creation and merging remain visually distinct.

Create and switch

git branch feature/example

Use this to prepare an isolated line of work before editing code.

Merge back

git merge main

Use this once the branch is ready to fold back into the shared line.

Remote

Remote commands are split to separate inspection from synchronization.

Set remote

git remote add origin "remote-url"

Use this when the local repository still needs a remote destination.

Inspect remotes

git remote -v

Use this to confirm which remote names and URLs the repository knows about.

Sync with main

git pull origin main

Use this when you need to bring in updates from the shared branch.

Publish changes

git push origin main

Use this after a commit is ready to move back to the shared remote.

Undo

Undo actions stay short so the page remains a quick reference instead of a tutorial.

Restore working tree

git restore file.txt

Use this when the change should be removed locally without rewriting history.

Undo last commit locally

git reset --soft HEAD~1

Use this when the last commit should be reopened without losing the staged state.

Record a reversal

git revert HEAD

Use this when the reversal itself should become a tracked commit.

Common Issues

Common checks are grouped as short entries so they read like quick notes during troubleshooting.

Quick checks

git status

Use this when you need to answer what changed, where you are, and what is pending.

Compare changes

git diff

Use this to inspect the exact edits before you decide the next step.

Confirm branch

git branch --show-current

Use this when branch confusion is the main thing you need to clear up.
