Placeholder Description

Each placeholder is shown as a small entry so the sidebar can jump straight to its meaning.

your-name

your-name

Placeholder for the author or contributor name used in local examples.

your-email

your-email

Placeholder for the email value that would normally be configured in Git identity settings.

origin

origin

Placeholder for the default remote name that points to the main repository location.

main

main

Placeholder for the primary branch name used as the central integration point.

remote-url

remote-url

Placeholder for the repository address that Git uses when talking to a remote server.

Repository start

Daily Commands

Everyday work stays in short entries so the page reads like a compact technical note.

Check state

git status
git add .
git commit -m "message"

Use this sequence to inspect changes, stage them, and record a local checkpoint.

Inspect history

git log --oneline

Use this when you only need a compact view of recent commits.

Branching

Branch operations are split into small entries so creation and merging remain visually distinct.

Create and switch

git branch feature/example
git switch feature/example

Use this to prepare an isolated line of work before editing code.

Merge back

git merge main

Use this once the branch is ready to fold back into the shared line.

Remote

Remote commands are split to separate inspection from synchronization.

Inspect remotes

git remote -v

Use this to confirm which remote names and URLs the repository knows about.

Sync with main

git fetch origin
git pull origin main
git push origin main

Use this sequence to bring in updates and then publish your own changes.

Undo

Undo actions stay short so the page remains a quick reference instead of a tutorial.

Restore working tree

git restore file.txt
git reset --soft HEAD~1

Use this when the change should be removed locally without rewriting the whole branch story.

Record a reversal

git revert HEAD

Use this when the reversal itself should become a tracked commit.

Common Issues

Quick checks

git branch --show-current

Use this when you need to answer what changed, where you are, and what is pending.

Conflict path

git diff
git add .
git commit

Use this as a reminder that conflicts usually become visible through the diff first.
